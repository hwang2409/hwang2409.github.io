Henry Wang writes technical notes about machine learning systems, CUDA, C++ frameworks, browser inference, research tooling, and small web backends.

The website is static, but the lab page connects to a small backend for live experiments. The backend runs FastAPI, exposes health checks, tokenization, and local next token scoring. The lab should avoid external model providers and use local models when possible.

CUDA backend notes. The main constraint is data movement. Kernel code matters, but keeping tensors on the device matters more. The CUDA backend has element wise kernels, activation kernels, BLAS calls through cuBLAS, reduction kernels, loss kernels, optimizer kernels, and convolution or batch normalization through cuDNN. Host to device copies add latency outside the kernel. A fast kernel does not matter if every operation starts with a host device copy. Persistent device tensors keep data resident on the GPU. Row major C++ data must be adapted to column major cuBLAS calls. The row major call swaps operands. Reductions need cooperation between threads. Softmax needs max, exponentiation, sum, and normalization. Large row softmax uses multiple kernel launches and temporary device buffers. Gradient clipping can introduce synchronization points when a scalar norm is copied from device to host. Memory pools reduce cudaMalloc and cudaFree overhead. cuDNN handles convolution and batch normalization because fast convolution kernels are a separate optimization problem. The largest remaining CUDA optimization is keeping data on the GPU in the first place.

Whitematter notes. Whitematter is a small deep learning framework written in C++. It implements tensors, autograd, layers, optimizers, CPU kernels, and GPU backends. The web UI generates C++ training programs, compiles them, runs them, and streams metrics back to the browser. A tensor is a contiguous float buffer with shape metadata, stride info, and a pointer to the operation that created it. Backward walks the graph in reverse topological order. Matmul backward produces grad times B transpose for A and A transpose times grad for B. ReLU backward masks negative input. Convolution backward is a transposed convolution. Every backward function is written by hand. SIMD handles element wise operations. BLAS handles GEMM. Metal and CUDA handle GPU operations. The training pipeline maps architecture JSON to a full C++ training script. Loss, accuracy, and learning rate stream to the browser over server sent events.

CIFAR classifier notes. The demo uses a ResNet 18 trained from scratch with Whitematter. The model has 11 million parameters, CUDA accelerated training, ONNX export, and browser inference through ONNX Runtime Web. ResNet 18 uses residual blocks with two 3 by 3 convolutions and a skip connection. The skip connection gives the gradient a shorter path through the network. Convolution is implemented as matrix multiplication through im2col on CPU and cuDNN on GPU. Batch normalization stores running mean and variance for inference. The backward pass computes gradients for input, gamma, and beta. Training uses stochastic gradient descent with momentum, weight decay, cosine annealing, and data augmentation. ONNX Runtime Web loads the model in the browser and runs WebAssembly inference. The training stack is version dependent, but the inference artifact is portable.

Stack notes. The default small AI project stack uses React in the frontend, FastAPI in the backend, SQLite and Redis for storage and background work, Claude behind a service layer when external models are needed, and server sent events for progress updates. FastAPI is useful for async network calls, Pydantic validation, and streaming responses. Claude should not touch the database directly. SQLite in WAL mode handles concurrent reads and low write volume workflows. Long running tasks such as transcription, batch analysis, and PDF processing should run outside the request cycle. Server sent events work well for one way progress updates. Client writes use regular POST requests. A single Dockerfile, nginx config, and deploy workflow are enough for the current project scale.

Marginalia notes. Marginalia finds, reads, and connects papers, blog posts, and notes. A workspace description seeds a knowledge base that later searches can reuse. Haiku handles extraction and Sonnet handles synthesis when external models are enabled, but deterministic local stages should be preferred where they are sufficient. Ingestion runs query decomposition, parallel search, deduplication, fetch and parse, section aware chunking, extraction, embedding, and storage. Hybrid retrieval combines vector search, BM25 full text search, and concept expansion. Reciprocal rank fusion combines rank positions from different retrieval systems. Wiki first retrieval checks generated summaries before raw chunks. Concept graphs connect sources, claims, prerequisites, and related concepts. Deterministic edge generation uses shared concepts, title similarity, and embedding similarity. ELK layouts keep graph components structured.

Neuronic notes. Neuronic treats notes as structured study data instead of plain documents. It extracts concepts, prerequisites, definitions, and review state. Flashcards and quizzes are views over the same graph. SM 2 scheduling applies to flashcards, notes, and concepts. Quiz results update the source note review state. Knowledge graph nodes track concepts and edges track prerequisite relationships. Gap detection compares prerequisites against known concepts. Search combines semantic similarity, keyword matching, and recency. Study modes include flashcards, quizzes, active recall, Feynman explanations, Socratic dialogue, and focus sessions.

Local model notes. A small n gram model can run locally without Anthropic, OpenAI, or any external inference API. The model tokenizes site text, counts continuations after one, two, and three token contexts, and backs off to shorter contexts when a sequence has not appeared. It returns top tokens, probabilities, entropy, and the context order that matched. This is not a frontier language model, but it is honest, fast, inspectable, and aligned with a static personal site that acts like a small technical lab.
